## 报告漏洞

如果您发现安全漏洞，请**不要**通过公开的GitHub Issues报告。

### 报告流程

1. 发送邮件至 security@example.com
2. 提供详细的漏洞描述
3. 包含复现步骤
4. 我们会在48小时内回复

### 报告内容

请包含以下信息：

- 漏洞类型
- 影响范围
- 复现步骤
- 潜在影响
- 修复建议（如有）

### 响应时间

- 确认收到：48小时内
- 初步评估：7天内
- 修复发布：30天内

## 安全最佳实践

### 部署安全

1. **使用HTTPS**
   - 配置SSL证书
   - 强制HTTPS重定向

2. **配置防火墙**
   - 限制端口访问
   - 使用安全组

3. **定期更新**
   - 更新依赖包
   - 更新Docker镜像

4. **日志监控**
   - 启用访问日志
   - 监控异常请求

### API安全

1. **使用API Key**
   - 生成唯一的API Key
   - 定期轮换Key

2. **速率限制**
   - 配置请求限制
   - 防止滥用

3. **输入验证**
   - 验证查询参数
   - 防止注入攻击

### 数据安全

1. **隐私保护**
   - 不记录用户查询
   - 不存储个人信息

2. **加密传输**
   - 使用HTTPS
   - 加密敏感数据
