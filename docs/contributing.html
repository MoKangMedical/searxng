Ways to Contribute

Report Bugs

Found something broken? Open an issue with steps to reproduce, expected vs actual behavior, and your environment details.

Suggest Features

Have an idea? Check the roadmap first, then open a feature request with use cases.

Add Engines

Know a search engine we're missing? Add it to settings.yml and submit a PR. See engine development guide below.

Improve Docs

Typos, unclear instructions, missing examples? Documentation PRs are always welcome and highly valued.

Write Code

Fix bugs, add features, improve performance. See development setup below.

Test & Review

Try the latest version, report issues, review pull requests. Every review counts.

Development Setup

1

Fork & Clone

# Fork on GitHub, then:
git clone https://github.com/YOUR_USERNAME/searxng.git
cd searxng
git remote add upstream https://github.com/MoKangMedical/searxng.git
2

Create Branch

git checkout -b feature/your-feature-name
# or
git checkout -b fix/issue-number-description
3

Make Changes

# Edit files
# Test locally:
cd ~/.docker/searxng
docker compose restart searxng
curl "http://localhost:8888/search?q=test&format=json"
4

Commit & Push

git add -A
git commit -m "feat: description of change"
# or: "fix: description", "docs: description", "chore: description"
git push origin feature/your-feature-name
5

Open Pull Request

Go to GitHub and open a PR. Fill in the template with:

  • What this PR does
  • Related issue number
  • Testing steps
  • Screenshots (if UI changes)

Commit Convention

We use Conventional Commits for clear history:

PrefixUsageExample
feat:New featurefeat: add Baidu Images engine
fix:Bug fixfix: JSON API 403 error
docs:Documentationdocs: update API reference
chore:Maintenancechore: update dependencies
style:Code stylestyle: fix indentation

test:Teststest: add search API tests
perf:Performanceperf: optimize cache lookup

Adding a New Search Engine

  1. Find the engine class in SearXNG source (or create a new one)
  2. Add configuration to config/settings.yml: 
    engines:
  - name: new_engine
    engine: new_engine
    shortcut: ne
    categories: [general]
    disabled: false
    timeout: 4.0
  3. Test: curl "http://localhost:8888/search?q=test&format=json&engines=new_engine"
  4. Submit PR with engine name, categories, and test results

Code Style

  • Python: Follow PEP 8, use type hints, docstrings
  • JavaScript: ES6+, no jQuery, use fetch API
  • HTML: Semantic markup, accessibility attributes
  • CSS: CSS variables, no !important, mobile-first
  • YAML: 2-space indentation, no tabs

Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone. Please:

  • Be respectful and constructive
  • Focus on what is best for the community
  • Show empathy towards other contributors
  • Give and receive feedback gracefully
