Quick Deploy (5 Minutes)

1

Clone Repository

git clone https://github.com/SearXNG Medical AI - 医疗AI智能搜索引擎/searxng.git
cd searxng
2

Run Installer

chmod +x install.sh
./install.sh

The installer handles: Docker check, directory setup, config copy, container start, health verification.

3

Verify

# Web UI
open http://localhost:8888

# JSON API
curl "http://localhost:8888/search?q=test&format=json" | python3 -m json.tool

# Container status
docker ps --filter "name=searxng"
Done! SearXNG is running with 30+ search engines, Redis cache, and health checks.

Manual Deployment

Step 1: Create Runtime Directory

mkdir -p ~/.docker/searxng/config
macOS: Docker Desktop cannot bind-mount ~/Desktop paths. Always use ~/.docker/searxng/.

Step 2: Copy Configuration Files

# Docker Compose
cp docker-compose.yml ~/.docker/searxng/

# SearXNG settings (30+ engines, Redis cache, JSON API)
cp config/settings.yml ~/.docker/searxng/config/

# Optional: Custom CSS/JS
cp config/custom.css ~/.docker/searxng/config/
cp config/custom.js ~/.docker/searxng/config/

Step 3: Configure settings.yml

Key settings to review:

# settings.yml
server:
  secret_key: "CHANGE_ME_TO_RANDOM_STRING"  # Generate: openssl rand -hex 32

search:
  formats: [html, json, csv, rss]  # Enable JSON API
  autocomplete: "baidu"             # "google" for international
  concurrent_requests_per_engine: 5
  max_request_timeout: 5.0

redis:
  url: "redis://redis:6379/0"

cache:
  expire: 86400  # 24h TTL

Step 4: Start Services

cd ~/.docker/searxng
docker compose up -d

# Watch startup logs
docker compose logs -f --tail=50

Step 5: Verify Health

# Check containers
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# Test web interface
curl -s -o /dev/null -w "HTTP: %{http_code}\n" http://localhost:8888/

# Test JSON API
curl -s "http://localhost:8888/search?q=test&format=json" | head -c 200

# Test Redis
docker exec searxng-redis redis-cli ping

Production Hardening

SSL/TLS with Nginx ▼

Add Nginx reverse proxy with Let's Encrypt SSL:

# Install certbot
apt install certbot python3-certbot-nginx

# Get certificate
certbot --nginx -d search.yourdomain.com

# Nginx config (nginx/nginx.conf)
server {
    listen 443 ssl http2;
    server_name search.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/search.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/search.yourdomain.com/privkey.pem;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    location / {
        proxy_pass http://searxng:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
Redis Authentication ▼ 
# Generate password
REDIS_PASS=$(openssl rand -hex 16)

# Add to docker-compose.yml redis service
command: redis-server --requirepass ${REDIS_PASS} --save 30 1

# Update settings.yml
redis:
  url: "redis://:${REDIS_PASS}@redis:6379/0"
Rate Limiting ▼

Enable application-level rate limiting:

# settings.yml
server:
  limiter: true

# config/limiter.toml
[botdetection.ip_limit]
# Max requests per IP per time window
link_token = true

For Nginx-level rate limiting, see Security Configuration.

Automated Backups ▼ 
#!/bin/bash
# backup.sh - Run daily via cron
BACKUP_DIR="$HOME/searxng-backups/$(date +%Y%m%d)"
mkdir -p "$BACKUP_DIR"

# Backup config
cp -r ~/.docker/searxng/config "$BACKUP_DIR/"

# Backup Redis data
docker exec searxng-redis redis-cli BGSAVE
docker cp searxng-redis:/data/dump.rdb "$BACKUP_DIR/"

# Backup docker-compose
cp ~/.docker/searxng/docker-compose.yml "$BACKUP_DIR/"

# Cleanup old backups (keep 30 days)
find "$HOME/searxng-backups" -mtime +30 -delete

echo "Backup completed: $BACKUP_DIR"

Add to crontab: 0 3 * * * /path/to/backup.sh

Monitoring Setup ▼

Health check script for monitoring:

#!/bin/bash
# monitor.sh - Check all services
check() {
    local name=$1 url=$2
    status=$(curl -s -o /dev/null -w "%{http_code}" "$url" 2>/dev/null)
    if [ "$status" = "200" ]; then
        echo "OK: $name"
    else
        echo "FAIL: $name (HTTP $status)"
        # Send alert (email, Slack, etc.)
    fi
}

check "SearXNG" "http://localhost:8888/"
check "Redis" "http://localhost:8888/search?q=test&format=json"

# Docker health
docker ps --filter "health=unhealthy" --format "{{.Names}}" | while read c; do
    echo "UNHEALTHY: $c"
done

Common Deployment Issues

Container keeps restarting: Check logs with docker logs searxng --tail 50. Usually caused by invalid settings.yml.
"operation not permitted" on macOS: Docker Desktop cannot mount ~/Desktop. Move config to ~/.docker/searxng/.
JSON API returns 403: Add search.formats: [html, json] to settings.yml. Use search.formats NOT server.formats.
Baidu returns CAPTCHA: Normal anti-bot behavior. Use Sogou or 360search as alternatives.

For more troubleshooting, see FAQ.
