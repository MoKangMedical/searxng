How do I install SearXNG with Docker?Setup ▼

The fastest way is using our one-click installer:

git clone https://github.com/SearXNG Medical AI - 医疗AI智能搜索引擎/searxng.git
cd searxng
chmod +x install.sh
./install.sh

Or manually with Docker Compose:

mkdir -p ~/.docker/searxng/config
cp docker-compose.yml ~/.docker/searxng/
cp config/settings.yml ~/.docker/searxng/config/
cd ~/.docker/searxng && docker compose up -d

Important: On macOS, Docker Desktop cannot bind-mount ~/Desktop paths. Always use ~/.docker/searxng/ for runtime.

What are the system requirements?Setup ▼

Minimum: 1 vCPU, 256MB RAM, 1GB disk, Docker 20.10+

Recommended: 2 vCPU, 512MB RAM, 5GB disk, Docker 24+

Supported OS: Linux (Ubuntu 20.04+, Debian 11+), macOS (with Docker Desktop), Windows (WSL2)

Network: Outbound HTTPS access to search engines. For China deployments, ghcr.io mirror is used automatically.

Why do I get "operation not permitted" on macOS?Troubleshoot ▼

Docker Desktop on macOS cannot bind-mount paths under ~/Desktop due to macOS sandboxing. Solutions:

  1. Move config to ~/.docker/searxng/config/
  2. Use named Docker volumes instead of bind mounts
  3. Add Docker Desktop to Full Disk Access in System Settings

Our install.sh script handles this automatically.

Why does the JSON API return 403 Forbidden?Troubleshoot ▼

The JSON API is disabled by default. To enable it, add to settings.yml:

search:
  formats:
    - html
    - json
    - csv
    - rss

Critical: The key is search.formats, NOT server.formats. SearXNG's webapp.py checks settings['search']['formats'] specifically. This was verified in source code.

After editing, restart: docker restart searxng

How do I add Chinese search engines (Baidu, Sogou)?Config ▼

Add to the engines section in settings.yml:

engines:
  - name: baidu
    engine: baidu
    shortcut: bd
    categories: [general]
    disabled: false
    timeout: 4.0

  - name: sogou
    engine: sogou
    shortcut: sg
    categories: [general]
    disabled: false
    timeout: 4.0

  - name: sogou wechat
    engine: sogou_wechat
    shortcut: sgw
    categories: [news]
    disabled: false
    timeout: 4.0

Our default config includes 30+ engines optimized for China. Use settings-medical.yml for healthcare-specific engine selection.

Can I set default language to Chinese?Config ▼

No. Setting search.default_lang or ui.default_locale to "zh-CN" or "zh-Hans-CN" causes a ValueError crash. This is a known SearXNG limitation.

Workarounds:

  • Users can manually select language in the UI
  • Use URL parameter: ?language=zh-CN
  • Change autocomplete to baidu: search.autocomplete: "baidu"
How does Redis caching work?Config ▼

Redis caches search results to avoid redundant engine requests:

redis:
  url: "redis://redis:6379/0"

cache:
  cache_server: "redis://redis:6379/1"
  expire: 86400  # 24 hours TTL

Key behaviors:

  • Cache key = normalized query + engine selection + language
  • Hit ratio typically 80-90% for common queries
  • 12x faster response on cache hits (147ms vs 1.8s)
  • Redis uses AOF persistence (save every 30s if 1+ key changed)
  • Memory limit: 256MB with LRU eviction
How do I use the search API?API ▼

Python:

import requests

response = requests.get("http://localhost:8888/search", params={
    "q": "medical AI",
    "format": "json",
    "engines": "google,bing,arxiv"
})
data = response.json()
for result in data["results"]:
    print(f"{result['title']}: {result['url']}")

JavaScript:

const response = await fetch(
    "http://localhost:8888/search?q=medical+AI&format=json"
);
const data = await response.json();
console.log(data.results);

See API Documentation for full reference.

How do I enable API authentication?API ▼

API key authentication is configured at the Nginx level:

# nginx.conf
location /search {
    # API key validation
    if ($http_api_key != "your-secret-key") {
        return 401;
    }
    proxy_pass http://searxng:8080;
}

# Rate limiting per API key
limit_req zone=api burst=10 nodelay;

For production, use JWT tokens or OAuth2 for more robust authentication.

Container keeps restarting. How to debug?Troubleshoot ▼

Step-by-step debugging:

# 1. Check container status
docker ps -a --filter "name=searxng"

# 2. View logs
docker logs searxng --tail 100

# 3. Common errors:
# - "ValueError: Invalid settings.yml" -> Check YAML syntax
# - "Address already in use" -> Port 8888 occupied
# - "Connection refused" -> Redis not running

# 4. Validate config
docker exec searxng cat /etc/searxng/settings.yml

# 5. Full rebuild
cd ~/.docker/searxng
docker compose down -v
docker compose up -d
Why does Baidu show "Suspended: CAPTCHA"?Troubleshoot ▼

Baidu's anti-bot detection triggers CAPTCHA after repeated queries. This is NOT a config error.

Workarounds:

  • Use Sogou or 360search for Chinese web search (more reliable)
  • Academic engines (Google Scholar, arXiv) never have this issue
  • WeChat search (sogou_wechat) is also reliable
  • Baidu may work intermittently after cooldown period
  • Reduce concurrent requests: concurrent_requests_per_engine: 2
Docker Desktop disconnects on macOSTroubleshoot ▼

Docker Desktop on macOS is known for instability. Fix sequence:

# Kill and restart Docker Desktop
osascript -e 'quit app "Docker"'
sleep 10
open -a Docker
sleep 60  # Wait for full startup
docker ps  # Verify

If this fails, check for backend process:

pgrep -f "com.docker.backend" || echo "Backend not running"

Consider using restart: unless-stopped in docker-compose.yml to auto-recover containers after Docker restarts.

How does medical search integration work?Medical ▼

The medical search module wraps SearXNG with domain-specific methods:

from ai.medical_search import MedicalSearchEngine

engine = MedicalSearchEngine("http://localhost:8888")

# Search drug information
results = engine.search_drug_info("metformin")
# Returns: baidu results, academic papers, WeChat articles

# Search rare diseases
results = engine.search_rare_disease("Gaucher disease")
# Returns: overview, genetics, patient resources, clinical trials

Integrates with PubMed, ClinicalTrials.gov, DrugBank, and medical knowledge bases via dedicated engine configurations.
